# Accessibility

## Required support

- VoiceOver
- Dynamic Type up to accessibility sizes
- Reduce Motion
- Increase Contrast
- Differentiate Without Color
- Bold Text
- haptic off switch
- sound off switch

## Guided session accessibility

- Every arousal level has a spoken label.
- Warning state uses text, shape, sound/haptic, not red alone.
- Controls remain at least 44 pt.
- No gestures are the sole method of input.
- Timer announcements are not overly frequent.

## Cognitive accessibility

- One decision per screen.
- Plain language.
- Consistent button placement.
- Avoid long paragraphs during an active session.
- Allow users to review instructions before starting.

## Motion safety

- No strobing.
- Red pulse frequency remains below problematic flashing ranges.
- Reduce Motion replaces ambient movement with static gradients.

## Privacy accessibility

Biometric failure must offer an accessible PIN entry path. Privacy cover must not trap VoiceOver focus.
